1. Lấy code mới nhất (Mỗi khi bắt đầu làm)
Trước khi code bất cứ thứ gì, hãy đảm bảo bạn đang có bản mới nhất từ Leader:

 Chạy trong terminal:
git checkout main
git pull origin main


2. Tạo nhánh riêng (Bắt buộc)
Tuyệt đối không code trực tiếp trên nhánh main. Mỗi người làm một nhiệm vụ thì tạo một nhánh riêng:

Ví dụ: git checkout -b feature/home-page (Nếu làm trang chủ)



3. Đẩy code lên để Leader duyệt
Sau khi làm xong tính năng và test chạy ổn trên máy cá nhân:

Chạy trong terminal:
git add .
git commit -m "Hoàn thành: [Tên tính năng bạn làm]"
git push origin [ten-nhanh-cua-ban]

Sau đó, lên GitHub nhấn nút "Compare & Pull Request" để Leader kiểm tra code và gộp vào bản chính.




Một số lưu ý quan trọng:
Hạn chế sửa file main.dart: Leader đã cài đặt sẵn Route. Bạn chỉ cần code trong đúng thư mục màn hình của mình.

Lỗi thư viện: Nếu thấy gạch đỏ ở các dòng import, hãy chạy lệnh flutter pub get ở Terminal.

Tên biến: Luôn ưu tiên dùng các hàm có sẵn trong CartProvider mà Leader đã viết (như addToCart, totalAmount).
